# No-Code Automation Workflow Project

## 프로젝트 소개

본 프로젝트에서는 노코드 자동화 도구인 **Make**와 **Zapier**를 활용하여 동일한 자동화 워크플로우를 구현하고 두 도구를 비교 분석하였다.

또한 반복적으로 수행하는 업무를 자동화하는 파이프라인을 직접 설계하고 구현하여 자동화 과정과 동작 원리, 그리고 확장 가능성 학습하였다.

---

# 프로젝트 1. 자동화 도구 비교 구현

## 자동화 주제(Google Form 과제 제출 알림 시스템)

Google Form 응답이 Google Sheets에 저장되면 제출 여부를 확인한 후 Discord 채널로 자동 알림을 전송하는 워크플로우를 구현하였다.

---

## 워크플로우 구조

```
Trigger: Google Form 제출

↓ (입력 데이터: 이름, 과목, 제출 여부)

Google Sheets 저장

↓

Filter (조건: 제출 여부 = "예")

↓

Action 1: Google Sheets 로그 저장 (처리 상태 기록)

↓

Action 2: Discord 메시지 전송 (최종 출력)

```

---

## 구글 시트 (입력/출력 데이터 구조)

Google Form 제출 데이터는 Google Sheets에 자동 저장되며 다음 정보를 포함한다.

- 이름 (String)
- 과목 (String)
- 제출 여부 (String: 예/아니오)
- 타임스탬프 (Timestamp)

## Make 구현

### 사용 모듈

| 단계 | 구성 |
|------|------|
| Trigger | Google Sheets - Watch New Rows |
| Filter | 제출 여부 = "예" |
| Action 1 | Google Sheets - Update Row (로그 저장) |
| Action 2 | Discord - Send Message |

### 자동 실행 증빙 (수동 개입 없음)

본 워크플로우는 Google Form 제출 이후 별도 실행 없이 자동으로 수행된다.

실행 흐름:

1. Google Form 제출 발생
2. Google Sheets 자동 저장
3. Make Trigger 자동 감지
4. Filter 자동 조건 판단
5. Discord 메시지 자동 전송 완료

-이 과정에서 사용자의 수동 실행은 존재하지 않는다.

# 동일성 설계 기준 (보완)

Make와 Zapier에서 동일한 워크플로우를 구현하기 위해 단순 기능 비교가 아니라 "데이터 흐름 구조 자체를 동일하게 유지"하는 것을 설계 기준으로 설정하였다.

구체적으로는 다음 3가지 기준을 적용하였다.

- 입력 이벤트(Trigger)는 Google Sheets Row 생성으로 통일
- 조건 분기(Filter)는 "제출 여부 = 예"로 동일하게 설정
- 출력(Action)은 "로그 저장 + Discord 알림"으로 동일 구조 유지

이를 통해 도구가 달라도 동일한 업무 로직이 유지되도록 설계하였다.

---

# Make와 Zapier 비교 분석 (업무 요구사항 기준)

|비교 항목|Make|Zapier|
|---|---|---|
|UI/UX|시각적인 노드 방식|단계별 리스트 방식|
|설정 난이도|초기 학습이 필요함|초보자도 쉽게 사용 가능|
|조건 분기|Filter 및 Router 지원|Filter 지원|
|실행 결과 확인|Scenario 실행 로그 제공|Task History 제공|
|무료 플랜|무료 사용량이 비교적 많음|무료 플랜 제한이 비교적 적음|

---

## Make 장단점

### 장점

- 시각적으로 워크플로우를 구성할 수 있다.
- 복잡한 자동화 및 조건 분기에 강하다.
- 다양한 서비스 연동이 유연하다.

### 단점

- 초기 학습 난이도가 있다.
- 구조 이해에 시간이 필요하다.

---

## Zapier 장단점

### 장점

- 직관적인 UI로 빠르게 설정 가능하다.
- 단순 자동화에 매우 적합하다.
  
### 단점

- 복잡한 로직 구성에 한계가 있다.
- 무료 플랜 제한이 크다.

---

## 비교 결과

복잡한 자동화나 다양한 조건 분기가 필요한 경우에는 Make가 적합하였고, 간단한 자동화를 빠르게 구축하는 경우에는 Zapier가 더 편리하였다.

# 프로젝트 2. 자유 주제 자동화 설계 및 구현

## 자동화 주제

Google Form 과제 제출 알림 자동화

---

## 반복 업무 정의

기존에는 과제 제출 이후 다음 작업을 수동으로 수행해야 했다.

- Google Form 제출 확인
- 제출 여부 확인
- Discord 채널 수동 알림 작성
- 누락 여부 체크

-이 과정은 반복적이며 실수 가능성이 있는 업무이다.

---

## 선정 도구

**Make**

### 선정 이유

- 시각적인 워크플로우 구성이 가능하다.
- Filter를 이용한 조건 분기가 쉽다.
- Google Sheets와 Discord 연동이 편리하다.

---

# 워크플로우 구조 (Trigger → Action → 분기 → 출력) 

```
Trigger: Google Sheets - Watch New Rows
(입력: 이름, 과목, 제출 여부)

↓

Filter: 제출 여부 = "예"
(조건: 데이터 무결성 확보 및 알림 대상 필터링)

↓

Action 1: Google Sheets 로그 저장
(입력 → 처리 상태 "완료" 기록)

↓

Action 2: Discord 메시지 전송
(출력: 최종 알림 메시지)

```

# Trigger / Action / Filter 정의

- Trigger: 외부 이벤트 발생 시 시스템 시작점  
  → (예: Google Form 제출 시 자동 실행 시작)

- Filter: 조건 기반 데이터 선별 단계  
  → (예: 제출 여부 = "예"인 데이터만 통과)

- Action: 실제 처리 및 결과 출력 단계  
  → (예: Google Sheets 저장, Discord 메시지 전송)
  
---

# 조건 분기 설계 기준 (업무 목적)

조건 "제출 여부 = 예"는 단순 필터가 아니라 다음 목적을 가진다:

- 잘못된 데이터 제거
- 알림 대상 최소화
- 운영 효율성 향상
- 중복 메시지 방지

---

# 자동 실행 증빙 (핵심 개선)

본 시스템은 완전 자동으로 실행되며 다음 과정으로 동작한다:

- Form 제출 → Sheets 저장 → Trigger 감지 → Filter 판단 → Discord 전송

- 전체 과정에서 수동 개입은 발생하지 않는다.

---

# 구조 확장 (모듈화)

조건이 증가할 경우 다음 방식으로 확장 가능하다:

- Router 기반 분기 처리
- 기능별 Scenario 분리
- 알림 / 저장 / 분석 모듈 분리

---

# 노코드 한계 및 코드 확장 (구체 예시 포함)

### 노코드 한계 업무 예시

- “대량 과제 제출 데이터 자동 채점 및 통계 분석”

- 이유:
- 복잡한 계산 로직 제한
- 데이터 분석 기능 부족
- 조건 기반 자동 점수 산출 어려움

### 코드 확장 방식

Python 기반 확장 시:

- pandas 데이터 분석
- API 기반 자동 점수 계산
- 조건별 자동 분류 처리

---

# 결과

Google Form 제출 후 Google Sheets에 자동 저장되며, 제출 여부가 "예"인 경우 로그 저장과 Discord 알림이 자동으로 실행되는 것을 확인하였다.

이 과정은 수동 개입 없이 완전히 자동으로 수행되었다.

---

# 느낀 점

이번 프로젝트를 통해 Trigger, Action, Filter 구조를 기반으로 한 자동화 설계의 중요성을 이해하였다.

또한 자동화 도구 선택은 단순 기능 비교가 아니라 협업, 운영, 디버깅, 확장성 같은 업무 요구사항 관점에서 이루어져야 한다는 점을 학습하였다.

노코드 자동화는 반복 업무를 효율적으로 줄이는 데 효과적이지만, 복잡한 데이터 처리 업무에서는 코드 기반 확장이 필요하다는 점도 확인하였다.
